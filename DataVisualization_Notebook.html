Week 10 Hands-On: Data Visualization

This notebook is designed for DESN2003 Week 10. The goal is to give students a short, editable workshop they can run in Google Colab without downloading extra files first.

By the end, students should be able to: - inspect a small dataset before plotting it - compare groups with box plots, histograms, and bar charts - look for patterns with a heatmap and scatter plot - create a quick qualitative visualization from transcript text - tweak a working example instead of starting from a blank page

The sample data is baked into the notebook. The notebook also writes out survey_data.csv and transcripts.json in the runtime so students can inspect or reuse them.

Before plotting anything, make sure students can answer basic questions: - What columns do we have? - Which columns are numeric, and which are categories? - How many students are in each group? - What does the text feedback actually sound like?

2. Quantitative Visualizations

These examples are deliberately small and editable. Students can change one line at a time and rerun the cell.

palette = {'GenAI': '#2A9D8F', 'Traditional': '#E76F51'}

3. Qualitative Visualizations

A word cloud is fast and approachable, but it is not the same thing as proper qualitative coding. It works well here as a low-friction classroom exercise.

transcript_df
all_text = ' '.join(item['text'] for item in transcripts)
custom_stopwords = STOPWORDS.union({'ai', 'chatgpt', 'using', 'used', 'tool', 'tools'})

wordcloud = WordCloud(
    width=900,
    height=450,
    background_color='white',
    stopwords=custom_stopwords,
    collocations=False,
).generate(all_text)

plt.figure(figsize=(12, 6))
plt.imshow(wordcloud, interpolation='bilinear')
plt.axis('off')
plt.title('Word Cloud Of Transcript Text')
plt.show()
theme_keywords = {
    'Helpful AI': ['helpful', 'tutor', 'great help', 'clear explanations', 'improved'],
    'Saves time': ['time', 'quickly', 'right away', 'saved'],
    'Trust / accuracy issues': ['confusing', 'wrong', 'double-checking', 'trust'],
    'Struggle without AI': ['without any ai', 'without ai', 'challenge', 'no one to ask', 'stressed'],
}

theme_hits = []
for item in transcripts:
    text = item['text'].lower()
    for theme, keywords in theme_keywords.items():
        if any(keyword in text for keyword in keywords):
            theme_hits.append({'TranscriptID': item['id'], 'Theme': theme})

theme_df = pd.DataFrame(theme_hits).drop_duplicates()
theme_summary = (
    theme_df['Theme']
    .value_counts()
    .rename_axis('Theme')
    .reset_index(name='Mentions')
)

plt.figure(figsize=(9, 5))
sns.barplot(data=theme_summary, x='Mentions', y='Theme', hue='Theme', dodge=False, palette='crest')
ax = plt.gca()
if ax.legend_:
    ax.legend_.remove()
plt.title('Simple Theme Count From Transcript Keywords')
plt.xlabel('Number of Transcripts Mentioning The Theme')
plt.ylabel('Theme')
plt.show()

display(theme_summary)

5. If Students Want To Use Their Own Data

In Google Colab, they can upload a CSV and replace survey_df with their own dataframe. The quickest path is:

from google.colab import files
uploaded = files.upload()
my_file = next(iter(uploaded))
survey_df = pd.read_csv(my_file)
survey_df.head()

After that, they can adapt the plotting cells above as long as the column names match what the chart expects.
